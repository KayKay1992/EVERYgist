Creating Dashboard Layout

## Dashboard Layout with Navbar — Step-by-Step Implementation

### Overview
You built a reusable admin dashboard layout that wraps all admin pages with a consistent navbar, collapsible side menu, and responsive design. The layout handles authentication checks, mobile menu toggling, and provides a clean structure for admin content.

### Files Involved

1. **Layout Component**
   - `frontend/gist_hub/src/components/layouts/DashboardLayout.jsx`

2. **Navbar Component**
   - `frontend/gist_hub/src/components/layouts/Navbar.jsx`

3. **SideMenu Component**
   - `frontend/gist_hub/src/components/layouts/SideMenu.jsx`

4. **Usage Examples**
   - `frontend/gist_hub/src/pages/Admin/Dashboard.jsx`
   - `frontend/gist_hub/src/pages/Admin/BlogPosts.jsx`
   - All other admin pages

5. **Dependencies**
   - `UserContext` for authentication state
   - `SIDE_MENU_DATA` from `utils/data.js` for menu items
   - React Router for navigation

### Step-by-Step Implementation

**Step 1: Created DashboardLayout Wrapper Component**
- File: `DashboardLayout.jsx`
- Accepts two props:
  - `children` — the actual page content to render inside the layout
  - `activeMenu` — string to highlight the current menu item
- Gets `user` from `UserContext` to check authentication
- Structure:
  - Always renders `<Navbar>` at the top
  - Conditionally renders content area only if `user` exists (logged in)
  - Uses flexbox layout: SideMenu on left, content area on right

**Step 2: Built the Navbar**
- File: `Navbar.jsx`
- Features:
  - Logo on the left
  - Hamburger menu icon (only visible on mobile/tablet with `block lg:hidden`)
  - Toggles `openSideMenu` state on click
  - Sticky positioning (`sticky top-0 z-30`) so it stays visible when scrolling
  - White background with subtle border
- Mobile behavior:
  - Shows/hides menu icon (HiOutlineMenu ↔ HiOutlineX)
  - When open, displays SideMenu in a fixed overlay at `top-[61px]`

**Step 3: Created SideMenu with Profile Section**
- File: `SideMenu.jsx`
- Profile Display (top section):
  - Shows user's profile image (circular, 80×80) or CharAvatar fallback (initials)
  - User's name and email below the avatar
  - Only renders if `user` exists
- Menu Items:
  - Maps over `SIDE_MENU_DATA` or `BLOG_NAVBAR_DATA` (depending on `isBlogMenu` prop)
  - Each item has an icon and label
  - Highlights active menu with gradient background (`from-sky-500 to-cyan-400`)
  - Clicking an item navigates and closes mobile menu
- Logout Button:
  - At the bottom of the menu
  - Clears localStorage, resets context, navigates to `/`
- Responsive:
  - Hidden on screens < 1080px in DashboardLayout
  - Shows in mobile overlay when hamburger is clicked

**Step 4: Responsive Layout Logic**
- Desktop (≥1080px):
  - SideMenu always visible on left
  - Navbar at top
  - Content area fills remaining space with `grow mx-5`
- Tablet/Mobile (<1080px):
  - SideMenu hidden by default (`max-[1080px]:hidden`)
  - Hamburger icon in Navbar
  - Clicking hamburger opens SideMenu as fixed overlay
  - Clicking menu item or outside closes it

**Step 5: Integrated into Admin Pages**
- Each admin page imports `DashboardLayout` and wraps its content:
  ```jsx
  <DashboardLayout activeMenu="Dashboard">
    <div className="p-6">
      {/* Page content here */}
    </div>
  </DashboardLayout>
  ```
- The `activeMenu` prop highlights the current section in SideMenu
- Examples: Dashboard, Blog Posts, Comments, Create/Edit Post all use this layout

**Step 6: Authentication Guard**
- DashboardLayout only renders the content area if `user` exists
- If not logged in, only the Navbar shows (though admin pages are also protected by PrivateRoutes)
- Double protection: route-level guard + layout-level check

### Key Features

**Navbar**
- Sticky header stays at top during scroll
- Mobile-friendly hamburger menu
- Logo branding
- Consistent across all admin pages

**SideMenu**
- User profile section with avatar/initials
- Icon-based navigation
- Active menu highlighting
- Logout functionality
- Responsive overlay on mobile

**Layout**
- Reusable wrapper reduces code duplication
- Consistent spacing and structure
- Flex-based responsive design
- Auth-aware rendering

### Styling Details

**Navbar**
- Height: `py-4 px-7` (padding)
- Background: `bg-white` with `border-b border-gray-200/50`
- Logo: `h-6 md:h-[26px]` (responsive height)
- Hamburger: `text-2xl`, toggles between menu/close icons

**SideMenu**
- Width: `w-45` (custom width)
- Height: `h-[calc(100vh-61px)]` (viewport height minus navbar)
- Profile avatar: `w-20 h-20 rounded-full`
- Menu items: `py-3 px-4`, active gets gradient background
- Sticky: `sticky top-[61px]` aligns below navbar

**Content Area**
- `grow` to fill available horizontal space
- `mx-5` horizontal margins

### Data Flow

1. User logs in → `UserContext` populated
2. Navigate to `/admin/dashboard`
3. PrivateRoutes checks role
4. Dashboard.jsx renders
5. DashboardLayout wraps content
6. DashboardLayout checks `user` from context
7. Renders Navbar + SideMenu + children
8. SideMenu shows profile and highlights "Dashboard" menu item
9. User clicks "Blog Posts" in menu
10. Navigate to `/admin/posts`, activeMenu changes, highlight updates

### Why This Approach

- **DRY Principle**: Single layout component used by all admin pages
- **Separation of Concerns**: Navbar, SideMenu, and Layout are independent, reusable components
- **Mobile-First**: Responsive design adapts to screen size
- **User-Centric**: Profile info and logout always accessible
- **Consistent UX**: All admin pages feel unified
