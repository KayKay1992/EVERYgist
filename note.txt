Creating SignUp Component

## SignUp Component Implementation Explanation

File: `frontend/gist_hub/src/components/Auth/SignUp.jsx`

Purpose
- Provide a registration form for new users to create accounts with optional admin access token for admin role assignment.

Key Features
1. Form Fields (Controlled Inputs)
   - `fullName`: User's full name
   - `email`: User email address
   - `password`: Account password
   - `adminAccessToken`: Optional field for admin registration (validated by backend)
   - `profilePic`: State reserved for future profile picture upload (not yet wired)

2. Validation
   - Checks fullName is not empty
   - Uses `validateEmail()` helper to verify proper email format
   - Validates password is provided
   - All checks happen client-side before API call

3. Context Integration
   - Uses `UserContext` to access `updateUser` (stores user + token) and `setOpenAuthForm` (closes auth modal)
   - Uses `useNavigate()` for post-signup redirection

4. Admin Role Logic
   - If user provides `adminAccessToken`, backend validates it against `ADMIN_ACCESS_TOKEN` env variable
   - Matching token grants `role: "admin"`, otherwise `role: "member"`
   - This allows both regular users and admins to register through the same form

5. API Call Flow (Currently Incomplete)
   - Form submit calls `handleSignUp`
   - Validates all required fields
   - Should POST to `API_PATHS.AUTH.REGISTER` with: `{ name: fullName, email, password, adminAccessToken }`
   - Backend returns `{ token, role, ...user }`
   - Calls `updateUser(response.data)` to save token + user to context/localStorage
   - Closes auth form and redirects based on role (admin → dashboard, member → home)

6. Error Handling
   - Displays inline error messages above the submit button
   - Handles both API errors (structured messages) and generic network errors

7. UI & Responsive Design
   - Two-column layout on medium+ screens (`md:grid-cols-2`)
   - Single column on mobile (`grid-cols-1`)
   - Form container: `w-[90vw]` on small screens, `w-[43vw]` on medium+
   - Side image (`AUTH_IMG`) hidden on small screens (`hidden md:block`)
   - Fixed height container (`h-[520px]`) keeps layout consistent

8. Switching to Login
   - "Already have an account? Log In" button calls `setCurrentPage('login')` to swap views in parent `AdminLogin` component

Files Involved
- `SignUp.jsx` (this component)
- `UserContext` for global auth state
- `axiosInstance` for HTTP requests
- `apiPaths.js` for endpoint constants
- `helper.js` for `validateEmail`
- `Input.jsx` reusable input component
- `AUTH_IMG` asset for visual design

Why This Approach
- Single form handles both regular and admin registration (no separate pages)
- Admin token is optional, so regular users leave it blank
- Reusable Input components keep code clean
- Responsive grid adapts to screen size
- Validation prevents unnecessary API calls
