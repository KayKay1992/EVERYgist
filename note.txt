PART 1: BLOG LANDING PAGE COMPONENT
================================================================================

üìÅ File: frontend/gist_hub/src/pages/Blog/BlogLandingPage.jsx

üéØ PURPOSE
----------
Serves as the public-facing blog homepage that displays all published blog posts
with pagination support. Features a prominent featured post at the top followed
by a responsive grid of additional posts.

üèóÔ∏è COMPONENT ARCHITECTURE
--------------------------

1. STATE MANAGEMENT
   - blogPostList: Array storing all fetched blog posts
   - page: Current page number for pagination
   - totalPages: Total number of available pages
   - isLoading: Loading state for API calls

2. DATA FETCHING STRATEGY
   - getAllPosts(pageNumber): Fetches paginated posts from backend
   - Initial load: Fetches page 1 on component mount (useEffect)
   - Load More: Appends new posts to existing list (spread operator)
   - API endpoint: GET /api/posts?status=published&page={pageNumber}
   - Response destructuring: { posts, totalPage }

3. PAGINATION IMPLEMENTATION
   - Smart state update: pageNumber === 1 ? posts : [...prevPosts, ...posts]
   - This replaces posts on initial load, appends on load more
   - Load More button visibility: page < totalPages
   - Disabled state during loading to prevent duplicate requests

4. LAYOUT STRUCTURE
   Grid System (responsive):
   - Desktop (md+): 12-column grid
     * Main content: 9 columns (col-span-9)
     * Sidebar space: 3 columns (for future sidebar)
   - Mobile: Single column (col-span-1)

5. POST RENDERING LOGIC
   Featured Post (First Item):
   - Conditionally renders if blogPostList.length > 0
   - Uses blogPostList[0] (array index 0)
   - Displays in full-width FeaturedBlogPost component
   - Prominent position with mb-8 spacing

   Other Posts (Remaining Items):
   - Uses blogPostList.slice(1) to skip first item
   - Renders in responsive grid:
     * Mobile: 1 column (grid-cols-1)
     * Tablet: 2 columns (md:grid-cols-2)
     * Desktop: 3 columns (lg:grid-cols-3)
   - 6-unit gap between cards

6. LOADING STATES
   Initial Loading:
   - Shows when: isLoading && blogPostList.length === 0
   - Displays: Spinning loader (LuLoaderCircle) centered
   - Style: Blue color, 4xl size, with py-20 vertical padding

   Load More Loading:
   - Replaces button icon with spinner
   - Changes button text to "Loading..."
   - Button remains disabled during fetch

7. EMPTY STATE
   - Shows when: !isLoading && blogPostList.length === 0
   - Message: "No blog posts found"
   - Centered with gray text color

8. DATA TRANSFORMATION
   - Date formatting: moment(updatedAt).format("MMM D, YYYY")
   - Author data: Nested object access (item.author.name, item.author.profileImageUrl)
   - Content preview: Full content passed (could be truncated in child components)

9. NAVIGATION HANDLING
   - handleClickPost(post): Navigates to /${post.slug}
   - Uses dynamic routing based on blog post slug
   - Slug-based URLs for SEO optimization

10. ERROR HANDLING
    - Try-catch block in getAllPosts
    - Logs errors to console
    - Finally block ensures loading state is cleared

üîß KEY TECHNICAL DECISIONS

   - Chose explicit prop passing for clarity

   - Default values: tags = [] prevents errors
   - onClick passed from parent for centralized navigation

4. PAGINATION STRATEGY
   - Infinite scroll alternative: Load More button
   - Benefits:
     * User control over loading
     * Reduced initial load time
     * Prevents scroll position issues
   - Implementation:
     * Track current page and total pages
     * Append new posts to existing array
     * Hide button when all pages loaded

5. DATE FORMATTING
   - Library: Moment.js
   - Format: "MMM D, YYYY" (e.g., "Nov 18, 2025")
   - Fallback: "-" if date unavailable
   - Ternary: updatedAt ? moment(...).format(...) : "-"

6. IMAGE HANDLING
   - Cover images: Object-cover maintains aspect ratio
   - Author avatars: Rounded-full with consistent sizing
   - Fallback: Could implement placeholder for missing images
   - Performance: Could add lazy loading for below-fold images

   - Slug-based URLs: SEO-friendly /${post.slug}
   - Tag filtering: /tag/${tag} for category pages
   - Client-side routing: React Router's useNavigate
   - No page reloads: SPA navigation

8. GRADIENT IMPLEMENTATION
   - Tailwind utility: bg-linear-to-r, bg-linear-to-br, etc.
   - Color stops: from-, via-, to- utilities
   - Transparency: /10, /20, /30, /50 opacity modifiers
   - Hover variants: group-hover: prefix for parent-triggered changes

9. ANIMATION PHILOSOPHY
   - Micro-interactions: Scale, translate, opacity
   - Duration hierarchy:
     * Quick: 300ms (buttons, icons)
     * Medium: 500ms (cards, overlays)
     * Slow: 700ms (image zoom)

   - GPU acceleration: Transform and opacity only
