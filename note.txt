Blog post editor page with markdown editor

==========================================
HOW I IMPLEMENTED BLOG POST EDITOR PAGE WITH MARKDOWN EDITOR
==========================================

OVERVIEW:
Created a blog post editor page where admins can write and edit blog posts using a markdown editor. The page includes fields for title, cover image, and content, with buttons to save as draft, publish, or delete posts.

==========================================
STEP 1: INSTALLED MARKDOWN EDITOR LIBRARY
==========================================

Installed the @uiw/react-md-editor package:
```bash
npm i @uiw/react-md-editor
```

This library provides a rich markdown editor with:
- Live preview of markdown
- Toolbar with formatting buttons
- Support for headings, bold, italic, links, images, lists, code blocks, etc.
- Easy to integrate with React

==========================================
STEP 2: SET UP COMPONENT STATE
==========================================

File: frontend/gist_hub/src/pages/Admin/BlogPostEditor.jsx

Created state to hold all post data:

```javascript
const [postData, setPostData] = useState({
  id: "",              // Post ID (for editing)
  title: "",           // Blog post title
  content: "",         // Markdown content
  coverImageUrl: "",   // Cover image file
  coverPreview: "",    // Preview URL for cover image
  tags: "",            // Tags (comma-separated)
  isDraft: "",         // Draft or published status
  generatedByAI: false // Flag if AI-generated
});
```

Why Single State Object?
- Easier to manage related data
- Single update function for all fields
- Simple to send to API

Created other states:
```javascript
const [error, setError] = useState("");           // Error messages
const [loading, setLoading] = useState(false);    // Loading state
const [openDeleteAlert, setOpenDeleteAlert] = useState(false); // Delete modal
```

==========================================
STEP 3: CREATED UPDATE FUNCTION
==========================================

Simple function to update any field in postData:

```javascript
const handleValueChange = (key, value) => {
  setPostData((prevData) => ({
    ...prevData,
    [key]: value,
  }));
};
```

How It Works:
- Takes a key (field name) and value (new value)
- Spreads previous data (...prevData)
- Updates only the specified field
- Example: `handleValueChange("title", "My Blog Post")`

STEP 4: BUILT THE EDITOR UI
==========================================

A) HEADER WITH ACTION BUTTONS:

```javascript
<div className="flex items-center justify-between">
  <h2>{isEdit ? "Edit Post" : "Add New Post"}</h2>
  <div className="flex items-center gap-3">
    {/* Delete button (only when editing) */}
    {isEdit && (
      <button onClick={() => setOpenDeleteAlert(true)}>
        <LuTrash2 /> Delete
      </button>
    )}
    
    {/* Save as Draft button */}
    <button onClick={() => handlePublish(true)}>
      <LuSave /> Save as Draft
    </button>
    
    {/* Publish button */}
    <button onClick={() => handlePublish(false)}>
      <LuSend /> {isEdit ? "Update & Publish" : "Publish Post"}
    </button>
  </div>
</div>
```

Features:
- **Conditional Title**: Shows "Edit Post" or "Add New Post" based on mode
- **Delete Button**: Only visible when editing existing post
- **Save as Draft**: Saves without publishing
- **Publish**: Publishes or updates the post
- **Loading State**: Disables buttons during API calls

B) POST TITLE INPUT:

```javascript
<div className="mt-4">
  <label>Post Title</label>
  <input
    type="text"
    value={postData.title}
    onChange={({ target }) => handleValueChange("title", target.value)}
    placeholder="Enter your blog post title"
  />
</div>
```

Simple text input connected to state via handleValueChange.

C) COVER IMAGE SELECTOR:

```javascript
<div className="mt-4">
  <CoverImageSelector
    image={postData.coverImageUrl}
    setImage={(value) => handleValueChange("coverImageUrl", value)}
    preview={postData.coverPreview}
    setPreview={(value) => handleValueChange("coverPreview", value)}
  />
</div>
```

Uses custom component to handle image upload and preview.

D) MARKDOWN EDITOR:

```javascript
<div className="mt-3">
  <label>Content</label>
  <div data-color-mode="light">
    <MDEditor
      value={postData.content}
      onChange={(data) => handleValueChange("content", data)}
      commands={[
        commands.bold,
        commands.italic,
        commands.strikethrough,
        commands.hr,
        commands.quote,
        commands.divider,
        commands.link,
        commands.code,
        commands.image,
        commands.unorderedListCommand,
        commands.orderedListCommand,
        commands.checkedListCommand,
        commands.heading,
        commands.heading1,
        commands.heading2,
        commands.heading3,
        commands.heading4,
        commands.heading5,
        commands.heading6,
      ]}
      hideMenuBar={true}
    />
  </div>
</div>
```

MDEditor Breakdown:
- **value**: Current markdown content from state
- **onChange**: Updates content state when user types
- **commands**: Toolbar buttons available (bold, italic, headings, links, etc.)
- **hideMenuBar**: Hides the top menu bar for cleaner look
- **data-color-mode="light"**: Sets light theme

==========================================
STEP 5: COVER IMAGE SELECTOR COMPONENT
==========================================

File: frontend/gist_hub/src/components/Inputs/CoverImageSelector.jsx

A) IMAGE UPLOAD HANDLING:

```javascript
const handleImageChange = (event) => {
  const file = event.target.files[0];
  if (file) {
    setImage(file);
    const preview = URL.createObjectURL(file);
    if (setPreview) {
      setPreview(preview);
    }
    setPreviewUrl(preview);
  }
};
```

What This Does:
1. Gets the selected file from input
2. Saves the file to state (for upload)
3. Creates a preview URL using `URL.createObjectURL()`
4. Sets preview for immediate visual feedback

B) UI STATES:

**Before Image Selected** (Empty State):
```javascript
<div onClick={onChooseFile}>
  <LuFileImage />
  <p>Click to select cover image</p>
</div>
```
- Shows placeholder with upload icon
- Clicking opens file picker

**After Image Selected** (Preview State):
```javascript
<div>
  <img src={preview || previewUrl} alt="Cover Preview" />
  <button onClick={handleRemoveImage}>
    <LuTrash />
  </button>
</div>
```
- Shows selected image preview
- Trash button to remove image

C) REMOVE IMAGE:

```javascript
const handleRemoveImage = () => {
  setImage(null);
  setPreviewUrl(null);
  if (setPreview) {
    setPreview(null);
  }
};
```

Clears all image-related state, returning to empty state.

==========================================
STEP 6: PAGE ROUTING & MODE DETECTION
==========================================

The component accepts an `isEdit` prop to determine mode:

```javascript
const BlogPostEditor = ({ isEdit }) => {
  const { postSlug } = useParams();
  
  useEffect(() => {
    if (isEdit) {
      fetchPostDetailsBySlug(); // Load existing post data
    } else {
      generatePostIdeas();      // Generate AI ideas for new post
    }
  }, [isEdit, postSlug]);
};
```

Two Modes:
1. **Create Mode** (`isEdit = false`):
   - Empty form
   - Button says "Publish Post"
   - Generates AI ideas on load

2. **Edit Mode** (`isEdit = true`):
   - Fetches existing post by slug
   - Pre-fills form with post data
   - Button says "Update & Publish"
   - Shows Delete button

==========================================
HOW THE MARKDOWN EDITOR WORKS
==========================================

1. **User Types**:
   - User types in editor (e.g., "# Heading")
   - MDEditor component detects change

2. **State Updates**:
   - onChange fires with new content
   - handleValueChange updates postData.content
   - Component re-renders with new value

3. **Live Preview**:
   - MDEditor shows formatted preview
   - "# Heading" appears as large heading
   - **Bold** appears bolded, etc.

4. **Toolbar Buttons**:
   - Click "Bold" button
   - Inserts `**text**` at cursor
   - Preview shows bold text

5. **Final Save**:
   - Click Publish
   - Markdown content sent to API
   - Stored in database as-is
   - Rendered on blog when viewing
