## API Paths Implementation Explanation (Frontend)

### Overview
You created a centralized API configuration file (`apiPaths.js`) that defines all backend endpoints in one place. This makes your frontend API calls consistent, easy to maintain, and prevents typos or hardcoded URLs scattered across components.

### Structure

**BASE_URL**
- `http://localhost:8000` - Your backend server address
- Change this to production URL when deploying (e.g., `https://api.yourapp.com`)

**API_PATHS Object**
Organized by feature area with clear comments for each endpoint:

1. **AUTH** - Authentication endpoints
   - `REGISTER`: `/api/auth/register` - User signup
   - `LOGIN`: `/api/auth/login` - User login
   - `GET_PROFILE`: `/api/auth/profile` - Fetch current user profile

2. **IMAGE** - Image upload
   - `UPLOAD_IMAGE`: `/api/auth/upload-image` - Upload profile image via multipart form

3. **DASHBOARD** - Admin dashboard
   - `GET_DASHBOARD_DATA`: `/api/dashboard-summary` - Fetch dashboard stats (posts, comments, views, etc.)

4. **AI** - AI-powered content generation
   - `GENERATE_BLOG_POST`: `/api/ai/generate` - Generate full blog post from title + tone
   - `GENERATE_BLOG_POST_IDEAS`: `/api/ai/generate-ideas` - Get 5 blog ideas for a topic
   - `GENERATE_COMMENT_REPLY`: `/api/ai/generate-reply` - AI-generated comment reply
   - `GENERATE_POST_SUMMARY`: `/api/ai/generate-summary` - Generate title + summary + key takeaways

5. **POSTS** - Blog post CRUD and features
   - `GET_ALL`: `/api/posts` - List all posts (supports ?status=published|draft&page=1)
   - `CREATE_POST`: `/api/posts` - Create new post
   - `UPDATE(id)`: `/api/posts/${id}` - Update post by ID (dynamic function)
   - `DELETE(id)`: `/api/posts/${id}` - Delete post by ID
   - `GET_BY_SLUG(slug)`: `/api/posts/slug/${slug}` - Get single post by slug
   - `GET_BY_TAG(tag)`: `/api/posts/tag/${tag}` - Filter posts by tag
   - `SEARCH`: `/api/posts/search` - Search posts by keyword
   - `GET_TRENDING_POSTS`: `/api/posts/trending` - Top 5 posts by views/likes
   - `INCREMENT_VIEW(id)`: `/api/posts/${id}/view` - Increment view count
   - `LIKE_POST(id)`: `/api/posts/${id}/like` - Like a post (requires auth)

6. **COMMENTS** - Comment management
   - `ADD(postId)`: `/api/comments/${postId}` - Add comment to a post
   - `GET_ALL_BY_POST(postId)`: `/api/comments/${postId}` - Get comments for a post
   - `GET_ALL`: `/api/comments` - Get all comments (admin view)
   - `DELETE(commentId)`: `/api/comments/${commentId}` - Delete comment by ID

### How to Use in Components

**Import in your components:**
```javascript
import { BASE_URL, API_PATHS } from '../utils/apiPaths';
```

**Example: Login request**
```javascript
const response = await axios.post(
  BASE_URL + API_PATHS.AUTH.LOGIN,
  { email, password }
);
```

**Example: Get post by slug**
```javascript
const slug = 'getting-started-with-react';
const url = BASE_URL + API_PATHS.POSTS.GET_BY_SLUG(slug);
const response = await axios.get(url);
```

**Example: Like a post**
```javascript
const postId = '507f1f77bcf86cd799439011';
const url = BASE_URL + API_PATHS.POSTS.LIKE_POST(postId);
await axios.post(url, {}, {
  headers: { Authorization: `Bearer ${token}` }
});
```

### Benefits

1. **Single Source of Truth** - All endpoints defined in one file
2. **Easy Updates** - Change BASE_URL or endpoint paths in one place
3. **No Typos** - Autocomplete helps catch mistakes
4. **Clear Documentation** - Comments explain each endpoint's purpose
5. **Dynamic Routes** - Functions like `UPDATE(id)` build paths with parameters
6. **Readable Code** - `API_PATHS.AUTH.LOGIN` is clearer than hardcoded strings

### Notes

- All endpoints match your backend routes exactly
- Dynamic functions (with `(id)` or `(slug)`) take parameters and return the full path
- Protected routes (like CREATE_POST, LIKE_POST) require JWT token in Authorization header
- When deploying, update BASE_URL to your production backend domain
