## Navbar Profile Info Component — Step-by-Step Implementation

### Overview
You created a small profile card that displays the logged-in user's name and profile picture in the navbar, along with a logout button. It appears only when the user is authenticated.

### Files Involved

1. **Component File**
   - `frontend/gist_hub/src/components/Cards/ProfileInfoCard.jsx`

2. **Integration**
   - `frontend/gist_hub/src/components/layouts/BlogLayout/BlogNavbar.jsx`

3. **Dependencies**
   - `UserContext` for user state and `clearUser` function
   - React Router's `useNavigate` for post-logout navigation

### Step-by-Step Implementation

**Step 1: Created the ProfileInfoCard Component**
- File: `ProfileInfoCard.jsx`
- Uses `useContext(UserContext)` to access:
  - `user` — the authenticated user object (contains name, profileImageUrl, role, etc.)
  - `clearUser` — function to clear user from context and localStorage
- Uses `useNavigate()` to redirect after logout

**Step 2: Component Structure**
- Conditionally renders only if `user` exists (returns `null` if not logged in)
- Layout:
  - Profile image (circular, 11×11, fallback to empty string if no image)
  - User's name (bold, 15px)
  - Logout button (sky-600 color, underline on hover)

**Step 3: Logout Logic**
- `handleLogout` function:
  1. Clears all localStorage (removes token and any cached data)
  2. Calls `clearUser()` to reset `UserContext` state
  3. Navigates to `/login` page

**Step 4: Integrated into BlogNavbar**
- File: `BlogNavbar.jsx`
- Imported `ProfileInfoCard` at the top
- Conditional rendering in the navbar's right section:
  - If `!user` → shows "Login/SignUp" button (opens auth modal via `setOpenAuthForm(true)`)
  - If `user` exists → shows `<ProfileInfoCard />` (hidden on mobile with `hidden md:block`)

**Step 5: User Flow**
1. User logs in → `updateUser` in context sets `user` state and stores token
2. BlogNavbar detects `user` is not null → hides login button, shows ProfileInfoCard
3. ProfileInfoCard displays user's profile image and name
4. User clicks "Logout" → clears storage, resets context, redirects to `/login`
5. BlogNavbar detects `user` is null again → shows "Login/SignUp" button

### Key Features

- **Responsive Design**: Only visible on medium+ screens (`hidden md:block`)
- **User Avatar**: Shows uploaded profile image or empty fallback
- **Clean Logout**: Clears all localStorage and context in one action
- **Navigation**: Auto-redirects to login page after logout

### Styling Details

- Profile image: `w-11 h-11` (44px circle), `bg-gray-300` fallback, `rounded-full`, `mr-3` spacing
- User name: `text-[15px]`, `font-bold`, `leading-3` (tight line height)
- Logout button: `text-sky-600`, `font-semibold`, `hover:underline`, `mt-2` spacing, `cursor-pointer`

### Why This Approach

- **Centralized State**: Uses `UserContext` so any component can access user data without prop drilling
- **Conditional Rendering**: Only shows when authenticated, keeping UI clean
- **Single Responsibility**: Small, focused component that only handles profile display and logout
- **Reusable**: Can be used in other navbars or headers if needed
