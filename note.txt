===========================================
MARKDOWN RENDERER COMPONENT IMPLEMENTATION
===========================================

1. OVERVIEW
-----------
The MarkdownContent component is a premium, highly-customized markdown renderer that transforms raw markdown text into beautifully styled HTML content. Built using react-markdown with remark-gfm for GitHub Flavored Markdown support, it provides a magazine-quality reading experience with custom styling for every markdown element.

Location: src/pages/Blog/components/MarkdownContent.jsx
Dependencies: 
  - react-markdown (v10.1.0): Core markdown rendering engine
  - remark-gfm (v4.0.1): GitHub Flavored Markdown plugin (tables, strikethrough, task lists)
  - react-icons/lu: Lucide icons for UI elements (LuCopy, LuCheck, LuCode, LuQuote)


2. CORE ARCHITECTURE
--------------------

Main Component Structure:
```
MarkdownContent (Main Component)
├── Props: { content: string }
├── Validation: Empty content check
└── ReactMarkdown Renderer
    ├── remarkPlugins: [remarkGfm]
    └── Custom component overrides for ALL markdown elements

CodeBlock (Nested Component)
├── Props: { code: string, language: string }
├── State: copied (boolean) for copy feedback
└── Features: macOS-style window, syntax highlighting placeholder, copy functionality
```

7. TECHNICAL IMPLEMENTATION DETAILS
------------------------------------

Component Mapping Strategy:
  - ReactMarkdown's `components` prop overrides default rendering
  - Each markdown element maps to custom component function
  - Functions receive props: node, children, className, href, src, alt, etc.
  - Return JSX with Tailwind classes for styling

Code Detection Logic:
```javascript
code({ node, className, children, ...props }) {
  const match = /language-(\w+)/.exec(className || "");
  const language = match ? match[1] : "";
  const isInline = !className;
  
  return !isInline ? (
    <CodeBlock code={...} language={language} />
  ) : (
    <code className="...inline-styles...">{children}</code>
  );
}
```
  - Regex extracts language from className (e.g., "language-javascript")
  - Inline code has no className
  - Block code renders CodeBlock component

Clipboard API:
```javascript
navigator.clipboard.writeText(code);
```
  - Modern browser API for copy functionality
  - Async operation (returns promise)
  - Triggers visual feedback via state update


Data Flow:
  1. Blog post fetched from API with markdown content
  2. Content passed as prop to MarkdownContent
  3. ReactMarkdown parses markdown string
  4. Custom components render styled HTML
  5. User sees beautifully formatted article

Content Source:
  - Backend stores markdown in database
  - Frontend receives raw markdown via API
  - No server-side rendering required
  - Client-side parsing with react-markdown
