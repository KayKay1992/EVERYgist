=========================================
SHARE POST COMPONENT IMPLEMENTATION
=========================================

1. OVERVIEW
-----------
The SharePost component is a social media sharing widget that enables users to share blog articles across multiple platforms. Built with React and styled using Tailwind CSS, it provides a clean, clinical interface with custom share buttons for Facebook, Twitter, LinkedIn, WhatsApp, TikTok, and a copy-to-clipboard functionality.

Location: src/pages/Blog/components/SharePost.jsx
Dependencies:
  - react-icons/lu: Lucide icons (LuCheck, LuCopy, LuShare2)
  - react-icons/fa: FontAwesome icons (FaFacebookF, FaTwitter, FaLinkedinIn, FaWhatsapp, FaTiktok)


2. COMPONENT ARCHITECTURE
--------------------------

Props:
  - title (string): The blog post title to include in share messages

State Management:
  - isCopied (boolean): Tracks whether the URL has been copied to clipboard
  - Uses useState hook for state management
  - Auto-resets copied state after 2 seconds

URL Handling:
  - shareUrl: Dynamically gets current page URL using window.location.href
  - SSR-safe with window check: typeof window !== 'undefined'
  - Ensures component works in both client and server environments

4. SHARE FUNCTIONALITY BREAKDOWN
---------------------------------

A. Facebook Share
-----------------
Handler: handleFacebookShare()
URL: https://www.facebook.com/sharer/sharer.php?u={URL}
Implementation:
  - Encodes shareUrl to handle special characters
  - Opens Facebook sharer dialog in popup window (600x400)
  - Standard Facebook Share API endpoint
Icon: FaFacebookF (Facebook "f" logo)
Styling: Blue-600 icon, blue-500 border/bg-blue-50 on hover


B. Twitter Share
----------------
Handler: handleTwitterShare()
URL: https://twitter.com/intent/tweet?url={URL}&text={TITLE}
Implementation:
  - Passes both URL and title as tweet text
  - URL encoding for both parameters
  - Opens Twitter compose dialog in popup
  - Pre-fills tweet with title and link
Icon: FaTwitter (Twitter bird)
Styling: Sky-500 icon, sky-500 border/bg-sky-50 on hover


C. LinkedIn Share
-----------------
Handler: handleLinkedInShare()
URL: https://www.linkedin.com/sharing/share-offsite/?url={URL}
Implementation:
  - LinkedIn's share-offsite endpoint
  - Automatically fetches page metadata
  - Opens LinkedIn sharing dialog
  - Professional network optimized
Icon: FaLinkedinIn (LinkedIn "in" logo)
Styling: Blue-700 icon, blue-600 border/bg-blue-50 on hover


D. WhatsApp Share
-----------------
Handler: handleWhatsAppShare()
URL: https://wa.me/?text={TITLE + URL}
Implementation:
  - Combines title and URL in single message
  - Opens WhatsApp Web or mobile app
  - Space-separated title and link
  - Universal WhatsApp sharing endpoint
Icon: FaWhatsapp (WhatsApp logo)
Styling: Green-600 icon, green-500 border/bg-green-50 on hover


E. TikTok Share
---------------
Handler: handleTikTokShare()
Implementation:
  - TikTok has NO direct web share API
  - Fallback: Copies URL to clipboard
  - Shows alert notification to user
  - User manually pastes in TikTok app
Technical Note: 
  - Uses Clipboard API: navigator.clipboard.writeText()
  - Alert provides user guidance
Icon: FaTiktok (TikTok logo)
Styling: Gray-900 icon, pink-500 border/bg-pink-50 on hover


F. Copy to Clipboard
--------------------
Handler: handleCopyClick()
Implementation:
  - Clipboard API: navigator.clipboard.writeText(shareUrl)
  - Promise-based async operation
  - Sets isCopied state on success
  - 2-second auto-reset timer
  - Error logging for clipboard failures

Visual Feedback:
  - Icon switches: LuCopy → LuCheck
  - Color changes: Gray-600 → Green-600
  - Conditional text message below buttons
  - Button title tooltip updates

State Flow:
  1. User clicks copy button
  2. Clipboard API writes URL
  3. isCopied set to true
  4. Icon and color change
  5. Success message appears
  6. After 2s, state resets to false


5. TECHNICAL IMPLEMENTATION DETAILS
------------------------------------

Window Object Safety:
```javascript
const shareUrl = typeof window !== "undefined" ? window.location.href : "";
```
Why: Prevents SSR (Server-Side Rendering) errors
- window object only exists in browser
- Check ensures code won't break during build/SSR
- Returns empty string as fallback

URL Encoding:
```javascript
encodeURIComponent(shareUrl)
encodeURIComponent(title)
```
Purpose: 
  - Escapes special characters in URLs and text
  - Prevents URL parameter injection
  - Ensures proper query string formatting
  - Handles spaces, ampersands, quotes, etc.

Window.open() Pattern:
```javascript
window.open(url, "_blank", "width=600,height=400")
```
Parameters:
  - url: Share platform endpoint
  - "_blank": Opens in new tab/window
  - "width=600,height=400": Popup dimensions
  - Creates centered dialog-style window

Clipboard API:
```javascript
navigator.clipboard.writeText(shareUrl)
  .then(() => { /* success */ })
  .catch((err) => { /* error */ })
```
Features:
  - Modern async/await compatible
  - Returns promise for error handling
  - Requires HTTPS in production
  - User permission may be required

13. TROUBLESHOOTING & COMMON ISSUES
------------------------------------

Issue: Icons not displaying
Solution: 
  - Verify react-icons package installed
  - Check import paths (react-icons/fa, react-icons/lu)
  - Ensure correct icon names (case-sensitive)
  - Clear Vite cache: Remove node_modules/.vite

Issue: Clipboard not working
Solution:
  - Ensure HTTPS in production
  - Check browser clipboard permissions
  - Verify modern browser version
  - Test in different browsers

Issue: Popups blocked
Solution:
  - User must allow popups for site
  - Test with popup blockers disabled
  - Consider fallback to new tab (_blank without dimensions)
  - Inform users to enable popups

Issue: Share URL incorrect
Solution:
  - Verify window.location.href returns correct URL
  - Check for SSR/build-time execution
  - Ensure component mounts on client only
  - Use useEffect if needed for URL
