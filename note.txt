READING TIME ESTIMATE FEATURE IMPLEMENTATION
===============================================

OVERVIEW
--------
Implemented an automatic reading time calculator that estimates how long it takes to read a blog post based on word count. This feature appears on both individual blog post pages and blog post summary cards across the application.

Reading Speed: 200 words per minute (industry standard)
Minimum Display: 1 minute (even for very short posts)


IMPLEMENTATION DETAILS
----------------------

1. CORE UTILITY FUNCTION
   Location: frontend/gist_hub/src/utils/helper.js
   Function: calculateReadingTime(content)

   How it works:
   a) Content Sanitization - Removes all markdown and HTML syntax for accurate word counting:
      - Code blocks (```...```)
      - Inline code (`...`)
      - Headers (# ## ### etc.)
      - Markdown formatting (* _ ~ [ ] ( ))
      - HTML tags (<div>, <p>, etc.)
      - Images (![alt](url))
      - Links ([text](url))

   b) Word Counting:
      - Splits sanitized text by whitespace
      - Filters out empty strings
      - Counts remaining words

   c) Calculation:
      - Divides word count by 200 (average reading speed)
      - Rounds up using Math.ceil()
      - Returns minimum of 1 minute for very short content

   Code snippet:
   ```javascript
   export const calculateReadingTime = (content) => {
     if (!content) return 1;
     
     const plainText = content
       .replace(/```[\s\S]*?```/g, "")      // Remove code blocks
       .replace(/`[^`]*`/g, "")              // Remove inline code
       .replace(/#{1,6}\s/g, "")             // Remove headers
       .replace(/[*_~[\]()]/g, "")           // Remove formatting
       .replace(/<[^>]*>/g, "")              // Remove HTML tags
       .replace(/!\[.*?\]\(.*?\)/g, "")      // Remove images
       .replace(/\[.*?\]\(.*?\)/g, "")       // Remove links
       .trim();

     const words = plainText.split(/\s+/).filter((word) => word.length > 0).length;
     const minutes = Math.ceil(words / 200);
     return minutes < 1 ? 1 : minutes;
   };
   ```

TECHNICAL DECISIONS
-------------------

1. Why 200 words per minute?
   - Industry standard for average adult reading speed
   - Used by Medium, Dev.to, and other popular blogging platforms
   - Accounts for comprehension, not just word recognition

2. Why remove markdown/HTML before counting?
   - Syntax characters don't count as "reading"
   - Code blocks are scanned differently than prose
   - Images and links don't contribute to reading time
   - Provides more accurate estimate for actual content

3. Why minimum 1 minute?
   - Prevents "0 min" or confusing fractional displays
   - Even short posts take time to load and comprehend
   - Matches user expectations and UX best practices

4. Why Math.ceil() instead of Math.round()?
   - Better to slightly overestimate than underestimate
   - Aligns with how other platforms calculate reading time
   - Users appreciate conservative estimates
