üì° DATA FETCHING STRATEGY
--------------------------

1. API INTEGRATION
   Function: getTrendingPosts()
   - Async/await pattern for clean asynchronous code
   - Endpoint: API_PATHS.POSTS.GET_TRENDING_POSTS
   - Method: GET request via axiosInstance
   - Response handling: response.data?.length > 0 ? response.data : []
   - Safety check: Optional chaining (?.) prevents undefined errors
   - Fallback: Returns empty array if no data

2. ERROR HANDLING
   - Try-catch block wraps API call
   - Errors logged to console for debugging
   - No user-facing error state (graceful degradation)
   - Empty state component handles no data scenario

3. LIFECYCLE MANAGEMENT
   - useEffect with empty dependency array []
   - Executes once on component mount
   - Cleanup function: return () => {} (placeholder for future cleanup)
   - Prevents unnecessary re-fetches during re-renders

üîß KEY DECISIONS
----------------

1. COMPONENT SEPARATION
   Why split into TrendingPostsSection + PostCard?
   - Reusability: PostCard could be used elsewhere
   - Maintainability: Easier to update individual parts
   - Testability: Can test components in isolation
   - Readability: Clearer code organization

2. DATA FETCHING IN COMPONENT
   - Self-contained: No props drilling from parent
   - Autonomous: Manages its own data
   - Flexible: Can be placed anywhere
   - Trade-off: Duplicates fetching logic if used multiple times

3. OPTIONAL CHAINING
   - tags[0]?.toUpperCase(): Prevents errors if tags undefined
   - response.data?.length: Safely checks for data
   - Defensive programming: Handles API inconsistencies

4. RANKING IMPLEMENTATION
   - Zero-based index from map
   - Display as index + 1 for user-friendly numbering
   - Simple, no state management needed
   - Assumes array order represents ranking

5. GRADIENT TECHNIQUE
   - bg-linear-to-r/br: Directional gradients
   - Multiple color stops: from-, via-, to-
   - Opacity modifiers: /20, /30, /50, /60
   - bg-clip-text: For gradient text effect

================================================================================
LESSONS LEARNED
================================================================================

üí° KEY TAKEAWAYS
----------------

1. COMPONENT DESIGN
   - Small, focused components are easier to maintain
   - Props interface should be explicit and minimal
   - Self-contained components reduce coupling
   - Composition over complex single components

2. STYLING APPROACH
   - Consistent design system creates cohesion
   - Gradient overlays add depth without clutter
   - Layered effects create premium feel
   - Animation timing is crucial for polish

3. USER EXPERIENCE
   - Sticky positioning enhances content discovery
   - Visual ranking provides social proof
   - Hover effects communicate interactivity
   - Empty states prevent confusion

4. PERFORMANCE
   - CSS animations outperform JavaScript
   - Functional components are lightweight
   - Proper key props optimize rendering
   - Image sizing prevents layout shift

5. DATA MANAGEMENT
   - Optional chaining prevents runtime errors
   - Fallback values handle edge cases
   - Single source of truth simplifies state
   - API integration should be isolated

6. VISUAL HIERARCHY
   - Size, color, and position guide attention
   - Consistent spacing creates rhythm
   - Gradient usage should be purposeful
   - Micro-animations enhance but don't distract

7. CODE ORGANIZATION
   - Separate concerns (container vs. card)
   - Descriptive variable names aid readability
   - Comments explain intent, not mechanics
   - Consistent patterns across codebase

8. ITERATION PROCESS
   - Start simple, add polish incrementally
   - Test hover states thoroughly
   - Verify empty and error states
   - Gather feedback early and often
