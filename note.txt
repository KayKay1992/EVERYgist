GENERATE BLOG POST COMPONENT - IMPLEMENTATION GUIDE
===================================================

OVERVIEW:
Form-based feature that creates full-length AI-generated blog posts (1500-2000 words) using Google's Gemini AI. Users provide a title and tone, and receive structured markdown content.

FRONTEND (GenerateBlogPostForm.jsx):

Props:
- contentParams: Prefilled data from AI idea cards (title, tone)
- setPostContent: Inserts generated content into editor
- handleCloseForm: Closes modal after generation

State:
- formData: { title, tone }
- isLoading: Loading state during API call
- error: Error messages

Handler (handleGenerateBlogPost):
- Validates title and tone
- POSTs to API_PATHS.AI.GENERATE_BLOG_POST
- On success: calls setPostContent(title, generatedPost), closes form
- On error: displays server message or generic fallback
- Always resets loading state

UI:
- Responsive (90vw mobile, 35vw desktop)
- Two inputs: Title, Tone (with placeholder examples)
- Submit button with spinner and disabled state
- Conditional error display

BACKEND (aiController.js):

Endpoint: POST /api/ai/generate (Private)

Process:
- Validates title and tone (400 if missing)
- Creates prompt: 1500-2000 words, markdown, engaging intro, subheadings, examples, conclusion
- Uses Gemini API (gemini-2.0-flash-lite)
- Returns markdown text
- Error handling with 500 status
