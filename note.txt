Blog Post SummaryCard

==========================================
HOW I IMPLEMENTED BLOG POST SUMMARY CARD
==========================================

OVERVIEW:
Created a reusable BlogPostSummaryCard component that displays blog post information in a compact, interactive card format. The card shows the post's cover image, title, last updated date, tags, views, likes, and a delete action button. It's used in the BlogPosts admin page to list all blog posts.

==========================================
STEP 1: COMPONENT STRUCTURE & PROPS
==========================================

File: frontend/gist_hub/src/components/Cards/BlogPostSummaryCard.jsx

Created a functional component accepting the following props:

```jsx
const BlogPostSummaryCard = ({
  title,        // Post title
  imgUrl,       // Cover image URL
  updatedOn,    // Last updated date (formatted)
  tags,         // Array of tag strings
  likes,        // Number of likes
  views,        // Number of views
  onClick,      // Handler when card is clicked (navigate to edit)
  onDelete,     // Handler when delete button is clicked
}) => {
  // Component logic
};
```

Key Features:
- Accepts all necessary post data as props
- Flexible and reusable for any blog post
- Callback functions for user interactions

==========================================
STEP 2: CARD LAYOUT & DESIGN
==========================================

A) MAIN CONTAINER:
```jsx
<div className="flex items-start gap-4 bg-white p-3 mb-5 rounded-lg cursor-pointer group" onClick={onClick}>
```

- Flexbox layout for horizontal arrangement
- White background with rounded corners
- Cursor pointer indicates clickability
- `group` class enables hover effects on children
- Clicking anywhere on card triggers onClick (navigation to edit page)

B) COVER IMAGE:
```jsx
<img
  src={imgUrl}
  alt={title}
  className="w-16 h-16 rounded-lg"
/>
```

- Fixed 16x16 size (64px x 64px)
- Rounded corners for modern look
- Positioned at left side of card

C) CONTENT SECTION:
```jsx
<div className="flex-1">
  <h3 className="text-[13px] md:text-[16px] font-medium text-black">{title}</h3>
  {/* Meta information */}
</div>
```

- `flex-1` makes it take remaining space
- Responsive text size (13px mobile, 16px desktop)
- Contains title and all metadata

==========================================
STEP 3: METADATA DISPLAY
==========================================

A) UPDATED DATE:
```jsx
<p className="text-sm text-gray-700 mb-2 font-medium bg-gray-100 px-2 py-1 rounded-md">
```

- Shows when post was last modified

- Visual separator between metadata sections

- Maps through tags array
- Each tag gets its own cyan badge

==========================================
STEP 4: DELETE BUTTON WITH HOVER EFFECT
==========================================

```jsx
<button 
  className="hidden md:group-hover:flex items-center gap-2 text-xs text-rose-500 font-medium bg-rose-50 px-3 py-1 rounded text-nowrap border border-rose-100 hover:border-rose-200 cursor-pointer"
  onClick={(e) => {
    e.stopPropagation();
    onDelete();
  }}
>
  <LuTrash2 className="" /> 
  <span className="hidden md:block">Delete</span>
</button>
```

Key Features:

A) VISIBILITY CONTROL:
- `hidden`: Hidden by default
- `md:group-hover:flex`: Shows on hover (desktop only)
- Only visible on desktop (md breakpoint)
- Cleaner interface, less clutter

B) STYLING:
- Rose/red theme for destructive action
- Trash icon (LuTrash2) for clear meaning
- Border changes on hover for feedback
- Text shows only on medium+ screens

C) EVENT HANDLING:
```jsx
onClick={(e) => {
  e.stopPropagation();
  onDelete();
}}
```

- `e.stopPropagation()`: Prevents card click event from firing
- Without this, clicking delete would also trigger navigation
- Calls onDelete callback to open confirmation modal

==========================================
STEP 5: INTEGRATION IN BLOGPOSTS PAGE
==========================================

File: frontend/gist_hub/src/pages/Admin/BlogPosts.jsx

Used the component in a map function to display all posts:

```jsx
{blogPostList.map((post) => (
  <BlogPostSummaryCard
    key={post._id}
    title={post.title}
    imgUrl={post.coverImageUrl}
    updatedOn={
      post.updatedAt
        ? moment(post.updatedAt).format("Do MMMM, YYYY")
        : "-"
    }
    tags={post.tags}
    likes={post.likes}
    views={post.views}
    onClick={() => navigate(`/admin/edit/${post.slug}`)}
    onDelete={() =>
      setOpenDeleteAlert({ open: true, data: post._id })
    }
  />
))}
```

Integration Details:

A) DATA MAPPING:
- Each post from API gets rendered as a card
- `key={post._id}` for React list optimization
- Moment.js formats the update date nicely

B) CLICK HANDLER:
```jsx
onClick={() => navigate(`/admin/edit/${post.slug}`)}
```

- Navigates to edit page when card is clicked
- Uses post slug for SEO-friendly URL
- Entire card is clickable for better UX

C) DELETE HANDLER:
```jsx
onDelete={() =>
  setOpenDeleteAlert({ open: true, data: post._id })
}
```

- Opens confirmation modal
- Passes post ID for deletion
- Prevents accidental deletions

==========================================
TECHNICAL DECISIONS EXPLAINED
==========================================

1. WHY STOP PROPAGATION?
   - Without it, clicking delete triggers both delete AND navigation
   - Ensures only the intended action occurs
   - Standard pattern for nested clickable elements

2. WHY GROUP HOVER?
   - Cleaner interface (no clutter)
   - Delete is destructive, shouldn't be too prominent
   - Discovered on hover maintains context
   - Mobile users tap directly on post to edit

3. WHY FLEXBOX LAYOUT?
   - Naturally responsive
   - Easy alignment control
   - flex-1 handles content stretching
   - Simple to maintain and understand

4. WHY SEPARATE BADGES?
   - Visual grouping of related data
   - Color coding provides instant recognition
   - Easier to scan than plain text
   - Modern, clean aesthetic

5. WHY FORMAT DATE WITH MOMENT.JS?
   - User-friendly format (e.g., "12th November, 2025")
   - Consistent date display across app
   - Easy to change format globally
   - Handles edge cases and localization
